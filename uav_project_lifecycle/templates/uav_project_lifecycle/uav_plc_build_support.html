Quality Control Testing System Review

The point of QC testing is to send out systems that are operational to a degree that the field team will not need to perform basic communications testing, will not need to troubleshoot internal rack wiring and can begin ‘hitting the ground running’ when firing up the system and beginning in field commissioning.

 

Each system that is fabricated in house goes through a quality control testing procedure. That procedure is ran by the Field Operations team under the QC phase task. It is important to work with the QC Engineering to ensure that any system critical, critical path or new technology based operations are tested and confirmed operational prior to the system shipping out.

 

The Project Engineer will run through the systems user interfaces and base line functionality to ensure that the system is shipping with functionality that both matches client expectations and Theory of Operations. Any functionality that does not function as required should be put on a punch list so that any possible preliminary work can be done to rectify the issue prior to the field installation and commissioning.

 

White Glove System Review

 

After a system is fully installed in the field and the field team is getting ready to leave from site for good the Project Fingering will perform an onsite White Glove Review.  This review must take place when the entire system is complete, not while it is being finished. The point of the White Glove Review is to ensure we are talking away from a completed install that is 100% inline with our USIS standards and that the system functions as expected.
